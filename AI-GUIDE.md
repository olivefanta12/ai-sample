# AI Guide

> 개발자의 역할은 코드를 직접 짜는 것이 아니라, **workflow를 잘 설계하는 것**이다.
> 명확한 workflow를 작성하고 AI에게 지시하는 것 — 이것이 앞으로의 개발 방식이다.
> 코드는 AI가 만든다. 개발자는 **무엇을, 왜, 어떤 순서로** 만들지를 정의한다.

---

## 1. 동기화

세션 시작 혹은 ai-guide읽어라고 할때, ./ai 폴더 내부에 있는 skill과 agent를 읽어서 이 문서의 Agent/Skill 테이블과 비교한다. 불일치(이름 변경, 추가, 삭제)가 있으면 **이 문서의 테이블을 실제 파일 기준으로 업데이트**한다.

## 2. 세션 시작 시 확인 사항

세션이 시작되면 아래의 Agent와 Skill 목록을 표 형태로 출력하여 현재 가용한 능력을 공유한다.

| 분류      | 이름                    | 트리거/용도            | 주요 능력                                  |
| :-------- | :---------------------- | :--------------------- | :----------------------------------------- |
| **Agent** | `codebase_investigator` | 코드베이스 분석        | 아키텍처 매핑, 의존성 파악, 루트 원인 분석 |
| **Agent** | `cli_help`              | 도움말                 | Gemini CLI의 특징 및 설정 안내             |
| **Agent** | `generalist`            | 대규모/반복 작업       | 배치 리팩토링, 고용량 데이터 처리, 조사    |
| **Skill** | `skill-creator`         | 스킬 생성              | 새로운 스킬 제작 가이드 제공               |
| **Skill** | `ajax-skill`            | `ajax`, `csr`, `fetch` | CSR 기반 부분 렌더링 스크립트 작성         |
| **Skill** | `task-interview-skill`  | `{task} 만들고 싶어`   | 요구사항 명세화 및 workflow 설계           |

---

## 3. 메모리에 로드할 규칙

아래 파일을 읽고 AI 메모리에 저장한다. 모든 코드 생성/수정 시 이 규칙을 따른다.

- `.ai/rule/common-rule.md` — 코드 컨벤션 규칙 (패키지 구조, 어노테이션 순서, 네이밍, 설정 등)
- `.ai/skill/{스킬명}-skill/skill.md` — 스킬들 (제목, 트리거만 읽기)
